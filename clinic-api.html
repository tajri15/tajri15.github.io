ClinicAPI - Full-Stack Doctor Booking System

Project Summary

ClinicAPI is a complete full-stack application for an online doctor booking platform. It features a robust backend REST API built with Java & Spring Boot, and a modern, interactive frontend built with React & TypeScript. The project covers the entire development lifecycle, from secure authentication and role-based authorization to automated testing and deployment with Docker, showcasing a production-ready architecture.

Technologies Used

  • Full-Stack: Java Spring Boot (Backend) & React + TypeScript (Frontend)
  • Authentication: Spring Security with JSON Web Tokens (JWT)
  • Database: PostgreSQL
  • Styling: Tailwind CSS
  • Testing: JUnit 5, Mockito (Backend) & H2 Database
  • Deployment: Docker, Docker Compose

Challenge & Solution

Challenge

The primary challenge was to build a secure, full-stack application from scratch, integrating a stateless Java backend with a modern React SPA. This involved implementing a robust JWT authentication flow, protecting endpoints based on user roles, handling real-time data fetching, and ensuring the entire multi-service application is portable and easy to run.

Solution

A secure layer was built using Spring Security with a custom JWT filter. The frontend manages authentication state globally using React Context. The entire stack (React frontend, Java backend, PostgreSQL database) was containerized using Docker and orchestrated with a single `docker-compose up` command, guaranteeing a seamless and replicable development environment.

Main Features

  • Full-Stack Architecture: Decoupled React frontend and Spring Boot backend.
  • Role-Based Authorization: Separate functionalities and UI for `ROLE_ADMIN` and `ROLE_PATIENT`.
  • Admin Dashboard: A dedicated UI for admins to perform CRUD operations on doctors and manage schedules.
  • Patient Experience: Public pages to view doctors/schedules, and protected pages for patients to book appointments and view their history.
  • Dynamic UI: The frontend UI changes dynamically based on the user's authentication status and role.
  • Professional Error Handling: A Global Exception Handler on the backend provides clean, consistent JSON error responses.
