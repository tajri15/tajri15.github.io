Attendance & Security Patrol System (SiAmalin)

Project Summary

Siamalin was my final project, an application specifically designed to modernize and digitize the daily operations of security officers on a university campus. This application focuses on improving accountability, reporting efficiency, and real-time monitoring of patrol activities. With a mobile-first interface, officers can easily access all essential features directly from their smartphones.

Technologies Used

  • Backend: PHP, Laravel Framework
  • Database: MongoDB
  • Frontend: Blade Template, JavaScript, Bootstrap 5
  • Key Features: Face-api.js (for face recognition), Leaflet.js (for GPS mapping), DeepFace (Python) for the backend service.

Main Features (Officer's Mobile App)

  • Officer Dashboard: The main page displays a summary of daily attendance, main navigation menus, and a monthly attendance recap.
  • Attendance with Face Verification: Officers clock in and out by scanning their faces. The system validates the face match and GPS location to ensure legitimate presence in the work area.
  • Start & Track Patrol: Officers can start a patrol session, which automatically tracks their movement route using GPS. The application also provides alerts if an officer moves outside the designated work radius.
  • Patrol & Attendance History: Officers can view their past patrol and attendance history, complete with details on time, duration, and route tracking on a map.
  • Digital Reporting: A feature to create and submit daily reports, activities, or incidents directly from the field. Reports can be supplemented with photographic evidence taken on the spot, along with the reporter's face verification.
  • Profile Management: Officers can view and update their profile data, including name, phone number, and password.
