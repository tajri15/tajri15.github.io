Go-Pulse - Real Time Uptime Monitoring System

Project Summary

Go-Pulse is a full-stack, real-time service monitoring application built from scratch to showcase modern development practices. The backend, built with Go (Golang), leverages concurrency to perform efficient, simultaneous health checks on multiple targets. Real-time status updates are pushed to a reactive Vue.js frontend via WebSockets, creating a seamless and interactive user experience. The entire application stack is containerized with Docker, ensuring easy and consistent deployment.

Technologies Used

  • Backend: Go (Golang), Gin Web Framework, Gorilla WebSocket
  • Frontend: Vue.js 3 (Composition API), Vite, Tailwind CSS
  • Database: PostgreSQL
  • Authentication: JSON Web Tokens (JWT)
  • Deployment: Docker, Docker Compose
  • Real-Time Communication: WebSockets

Challenge & Solution

Challenge

The core challenge was to design a system capable of performing numerous network checks concurrently without blocking the main application, while simultaneously pushing status updates to multiple users in real-time. This required a robust solution for managing background tasks and maintaining persistent, user-specific connections.

Solution

A concurrent worker pool was implemented in Go using goroutines and channels to handle the health checks efficiently. For real-time updates, a WebSocket "Hub" was created to manage the lifecycle of client connections. When a worker completes a check, it sends the result to the Hub, which then broadcasts the update to the specific user connected, ensuring instant feedback on the frontend dashboard.

Main Features

  • Multi-Type Monitoring: Users can monitor services via HTTP status, specific keyword presence on a webpage, or raw TCP port connection.
  • Real-Time Dashboard: The frontend updates automatically with status changes, response times, and last-checked data without needing a page refresh.
  • Secure Authentication: A complete JWT-based authentication flow (register, login, logout) protects all user-specific endpoints.
  • Historical Data Visualization: Interactive charts display the response time history for each monitor, with selectable time ranges (24h, 7d, 30d).
  • Full CRUD Functionality: Users can securely add, view, and delete their own monitoring targets.
  • Containerized Environment: The entire application (Go backend, PostgreSQL DB, Vue frontend) runs in a portable and isolated environment managed by Docker Compose.
