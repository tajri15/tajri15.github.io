CampusHub - Event Management System

Project Summary

CampusHub is a full-stack, real-time campus event management platform built with a microservices-style architecture. The project demonstrates the integration of two distinct technology stacks: a robust backend API powered by Java (Spring Boot) and a dynamic frontend user interface built with PHP (CodeIgniter). The entire application, including the MySQL database, is fully containerized with Docker and orchestrated with Docker Compose, ensuring a consistent and easily replicable development and deployment environment.

Technologies Used

  • Backend: Java, Spring Boot, Spring Web, Spring Data JPA, WebSockets
  • Frontend: PHP, CodeIgniter 4, Bootstrap 5
  • Database: MySQL
  • Deployment: Docker, Docker Compose

Challenge & Solution

Challenge

Integrating two different programming languages and frameworks (Java & PHP) into a cohesive application, and managing the multi-service environment for both development and deployment.

Solution

Designed a microservices-style architecture where the frontend communicates with the backend via a REST API. Utilized Docker Compose to define, manage, and link all services (frontend, backend, database), which simplified the complex setup into a single command, ensuring environmental consistency and easy onboarding for developers.

Main Features

  • Admin CRUD: Full functionality for admin users to Create, Read, Update, and Delete event data.
  • User Authentication & Roles: A complete manual authentication system (Register, Login, Logout) with Role-Based Access Control distinguishing between 'Admin' and 'User' privileges.
  • Event Registration: Authenticated users can register for events, with the backend validating for available quota and duplicate registrations.
  • View Registrants: Admins can view a list of all users registered for a specific event.
  • Real-Time Q&A: A dedicated page for each event featuring a live chat room, built with WebSockets, allowing for real-time interaction.
  • Interactive Map: Event locations are displayed on a dynamic map using Leaflet.js.
  • Dockerized Environment: The entire application stack is containerized, allowing it to run anywhere with a single `docker-compose up` command.
